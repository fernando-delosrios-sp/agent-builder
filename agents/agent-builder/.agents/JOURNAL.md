# JOURNAL — Critical Learnings Only

Your journal is NOT a log. Only add entries for CRITICAL architecture and engineering learnings.

## When to add an entry

Add journal entries only when you discover:
- An architecture or design pattern that caused a bug, regression, or rework
- A fix that had unexpected side effects, breaking changes, or cascading impact
- A rejected approach with important constraints to remember
- A surprising gap in the project's architecture, data flow, or dependency graph
- A reusable pattern, convention, or guardrail worth encoding for future work

Do NOT journal routine work such as:
- "Fixed a bug in X"
- "Added a new component/feature"
- Generic language or framework best practices
- Straightforward fixes without unique or project-specific insights

## Entry format

```
## YYYY-MM-DD - [Title]
**Discovery:** [What you found]
**Learning:** [Why it existed / root cause]
**Prevention:** [How to avoid next time]
```
